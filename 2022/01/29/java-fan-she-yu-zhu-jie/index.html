Java反射与注解

Java
Programming
发布日期:   2022-01-29

Java内存分析

java内存分为两个区域：堆和栈，除此之外，还有在堆内的方法区

  • 堆：存放new的对象和数组，可以被所有的线程共享，不会存放别的对象引用
  • 栈：存放基本变量类型（包含具体数值），引用类型的变量（存放这个引用在堆中的地址）
  • 方法区：位于堆内，包含所有class和static变量

类的加载过程

加载->链接->初始化

加载

将class文件字节码加载到内存里，将静态数据转换成方法区的运行时数据结构，然后生成一个代表这个类的java.lang.Class对象

链接

将Java类的二进制代码合并到JVM的运行状态中的过程

  1. 验证：确保加载的类信息符合JVM规范，没有安全方面的问题
  2. 准备：为类变量（static）分配内存并设置初始值，在方法区内分配内存存储
  3. 解析：虚拟机常量池内的符号引用（常量名）替换为直接引用（地址）的过程

初始化

  1. 执行类构造器<clinit>()方法（class init）的过程。编译器收集类中所有static变量的赋值动作和static代码块中的语句并将其合并，由类构造器合并执行。（类构造器是Class对象的构造器，不是这个类的构造函数）
  2. 当初始化一个类的时候，如果发现这个类的父类还没有初始化，就先要触发父类的初始化
  3. 虚拟机会保证一个类的<clinit>()方法在多线程环境中正确加锁和同步。

什么时候会发生类初始化？

类的主动引用一定会发生类初始化

  • 虚拟机启动时先初始化main方法所在的类
  • new一个类的对象
  • 调用类的静态成员和静态方法（除了final常量，因为它在常量池内）
  • 用java.lang.reflect中的方法对类进行反射调用
  • 当初始化一个类时，如果父类没有被初始化，则会先初始化父类

类的被动引用不会发生类初始化

  • 访问一个静态域时，只有真正声明这个域的类才会被初始化。eg：当通过子类引用父类的静态变量，不会导致子类的初始化
  • 通过数组定义类引用，不会发生类初始化（东西还没放进去）
  • 调用final常量（常量在链接阶段存入常量池中）

类加载器

类加载器是用来把class装载进内存的，JVM规范定义了这些类加载器：

引导类加载器BootstrapClassLoader

用C++编写的，JVM自带的类加载器，负责Java平台核心库，用来装载核心类库。这个加载器不能直接获取，即用SystemClassLoader.getParent().getParent()方法获取到的超类加载器为null

扩展类加载器ExtClassLoader

负责把jre/lib/ext目录下或者-D java.ext.dirs的jar包装入工作库。

系统类加载器AppClassLoader/SystemClassLoader

负责java -classpath或者-D java.class.path所指向的目录下的类和jar包装入工作，是最常用的加载器。

要获得系统类加载器可以加载的路径，可以打印System.getProperty("java.class.path")。

双亲委派机制

双亲委派机制用于防止同名包、类与 jdk 中的相冲突，实际上加载类的时候，先通知 appLoader，看 appLoader 是否已经缓存，没有的话，appLoader 又委派给他的父类加载器(extLoader)询问，看他是不是能已经缓存加载,没有的话，extLoader 又委派他的父类加载器(bootstrapLoader)询问，BootstrapLoader看是不是自己已缓存或者能加载的，有就加载，没有再返回 extLoader，extLoader 能加载就加载，不能的话再返回给 appLoader 加载，再返回的路中，谁能加载，加载的同时也加缓存里。因为其不停地找父类加载器，所以叫Parents加载机制，即双亲委派机制。因此不能自己定义java.lang.String类等等。

注解

是什么？

注解从JDK5.0开始使用，它不是程序本身，可以对程序做出解释（和注释的作用一样），但它还可以被其他程序（比如IDE）读取

注解的格式

注解以”@+名字”存在于代码块中，还可以添加一些参数。比如

@SuppresWarnings(value="unchecked")

在哪里使用？

可以附加在package、class、method、field上，相当于给这些东西添加了额外的辅助信息，可以通过反射实现对这些信息的访问。

内置的注解

@Override

定义在java.lang.Override中，这个注解只是用来修饰方法的，表示一个方法声明打算重写超类中的另一个方法声明。

@Deprecated

定义在java.lang.Deprecated中，这个注解用于修饰方法、属性、类，表示修饰的东西已经被废弃了，不鼓励使用。如果使用了被它修饰的东西，IDE会有一个Warning。

@SuppressWarnings

定义在java.lang.SuppressWarnings中，用来抑制编译时的警告信息（如“这个变量从来没有被使用过”），需要添加参数才能使用，这些参数都是枚举类型的键值对。

  • @SuppressWarnings(“all”)
  • @SuppressWarnings(“unchecked”)
  • @SuppressWarnings({“unchecked”, “deprecation”})
  • …

元注解

和“基本数据类型”类似，元注解是最基础的注解，负责注解其他注解。Java定义了4个标准的meta-annotation类型，被用来提供对其他annotation的说明。

这些类型和它们支持的类在java.lang.annotation包中可以找到

  • @Target：用于描述注解的使用范围

  • @Retention：表示需要在什么级别保存这个注释信息，用来描述注解的生命周期
    (SOURCE < CLASS < RUNTIME)

  • @Document：说明这个注解将被包含在javadoc中

  • @Inherited：说明子类可以继承父类中的该注解

Target

public @interface Target {
	ElementType[] value();
}

虽然看着很像方法，但是value其实是使用这个注解时传入的参数，默认用value，如果改成其他名字，就要指定了。

@Target可以附加参数，这些参数以枚举类型存储在ElementType.java文件中。有：TYPE、FIELD、METHOD、PARAMETER（参数）、CONSTRUCTOR、LOCAL_VARIABLE、ANNOTATION_TYPE、PACKAGE、TYPE_PARAMETER、TYPE_USE。

Retention

传入RententionPolicy类下的枚举类型来确定注解的生命周期。有：SOURCE、CLASS、RUNTIME。

自定义注解

@interface用来声明一个注解，格式：

public @interface 注解名 {
	...
}

在内容中写很多方法，其中每个方法其实是声明一个配置参数，方法的名字就是参数的名字，返回值就是参数类型。返回值只能是基本类型、Class、String、enum。可以通过default来声明参数的默认值。注解元素必须要有值，经常使用空字符串或者0作为默认值。

用@interface来定义注解时，自动继承java.lang.annotation.Annotation接口

@Target({ElementType.TYPE, ElementType.METHOD})
@Rentention(RetentionPolicy.RUNTIME)
public @interface MyAnnotation {
	String name() default "";
	int age() default 0;
	int id() default -1;
    String[] schools();
}
@MyAnnotation(name = "yhz", schools = {"FZU", "FZSZ"}) //不给school赋值就会报错
public void test(){ }

如果只有一个参数成员，一般参数名为value。

public @interface MyAnnotation {
	String value();
}

这样一来引用的时候可以不写value=

@MyAnnotation("yhz")
public void test(){ }

反射

Reflection，即反射是指程序在运行期可以拿到一个对象的所有信息。

反射的功能

  • 在运行时判断对象所属的类
  • 在运行时构造任意一个类的对象
  • 在运行时判断任意类具有的成员变量和方法
  • 在运行时获取泛型信息
  • 在运行时调用任意对象的成员变量和方法（private也能用！）
  • 在运行时处理注解
  • 生成动态代理
  • …

反射的优缺点

优点：

可以实现动态创建对象和编译，增加灵活性

缺点：

对性能有极大影响

反射相关API

  • java.lang.Class：类对象
  • java.lang.reflect.Method：类方法
  • java.lang.reflect.Field：类字段
  • java.lang.reflect.Constructor：类构造器
  • …

Class类

加载完类之后，在堆内存的方法区里就产生了一个Class类型的对象（一个类对应一个Class对象），这个对象包含了完整的类的结构信息。

在Object类中定义了getClass()方法，被所有类继承：

public final Class getClass

这个方法返回值的类型是一个Class类，这个类是反射的起源。反射也就是由对象反射出类本身。

获取Class对象

已知具体的类

可以通过类的class属性获取Class对象：

Class class = Person.class;

已知类的实例

可以通过调用getClass()方法获取Class对象：

Class class = person.getClass();

已知类的全名

可以通过Class类的静态方法forName()获取：

Class class = Class.forName("MyZoo.Animal");

获取内置类型

可以直接用TYPE属性：

Class class = Integer.TYPE;

获取父类类型

用getSuperclass()方法：

Class c1 = Student.class;
Class c2 = c1.getSuperclass();

哪些类型有Class对象？

  • class：所有类
  • Class类本身
  • interface
  • []：数组
  • enum：枚举
  • annotation：注解@interface
  • 基本数据类型
  • void

获取类的运行时结构

Field，Method，Constructor，Superclass，Interface，Annotation。即：

  • 类本身信息

  • 全部字段

  • 全部方法

  • 全部构造器

  • 所继承的父类

  • 实现的全部接口

  • 注解

先定义一个User类：

public class User {
    private int id;
    private int age;
    private String name;

    public User(int id, int age, String name) {
        this.id = id;
        this.age = age;
        this.name = name;
    }

    public User() {
    } //newInstance默认用无参构造器，不写就会出错或者必须加参数获得有参构造

    public String getName() {
        return this.name;
    }

    public void setName(String name) {
        this.name = name;
    }

    private void hello() {
    }
}

然后来试一试（这些方法都带有抛出异常）：

类名

用getName()获得带包名的类名，getSimpleName()获得类名。

System.out.println(c1.getName());
System.out.println(c1.getSimpleName());

以String类为例，会打印：

String

字段

用getFields()和getDeclaredFields()获取全部的字段，以User类为例：

for (Field field : c1.getFields()) {
	System.out.println(field);
}
for (Field declaredField : c1.getDeclaredFields()) {
	System.out.println(declaredField);
}

上面的for循环什么都不会打印，因为User类没有public属性。要想获取到私有的东西，要加一个Declared。

下面的for循环会输出：

用getDeclaredField(name)获取指定字段：

Field name = c1.getDeclaredField("name");
System.out.println(name);

就会输出第三行。

System.out.println(getName);

构造器

和方法差不多，把method换成constructor就行，需要附加参数，不加参数就是无参构造器。

Constructor declaredConstructor = c1.getDeclaredConstructor(int.class, int.class, String.class);
System.out.println(declaredConstructor);

输出：

public User(int,int,java.lang.String)

也可以用getDeclaredConstructors()获取全部构造器。反射可以通过获得私有构造器破坏单例模式。

动态创建对象执行方法

setAccessible

反射甚至可以直接无视private的限制！Method、Field、Constructor对象都有setAccessible方法。它是安全检查的开关。参数值为true则指示反射的对象在使用时取消权限检查。

  • 它能提高反射的效率，如果代码里必须用反射，而这一句代码需要频繁使用，就应该设置为true。
  • 它让原本无法访问的私有成员也可以访问。

下面是一个效率测试，对直接访问，普通的通过反射访问，关闭检查的通过反射访问（这里访问的也是public方法）各执行一亿次：

import java.lang.reflect.InvocationTargetException;
import java.lang.reflect.Method;

public class Test01 {
    public static void main(String[] args) throws InvocationTargetException, NoSuchMethodException, IllegalAccessException {
        test01();
        test02();
        test03();
    }

    public static void test01() {
        User user = new User();
        long starttime = System.currentTimeMillis();
        for (int i = 0; i < 1000000000; i++) {
            user.getName();
        }
        long endtime = System.currentTimeMillis();
        System.out.println("正常调用方法: " + (endtime - starttime) + "ms");
    }

    public static void test02() throws NoSuchMethodException, InvocationTargetException, IllegalAccessException {
        User user = new User();
        Class c1 = user.getClass();
        Method getName = c1.getMethod("getName", null);
        long starttime = System.currentTimeMillis();
        for (int i = 0; i < 1000000000; i++) {
            getName.invoke(user, null);
        }
        long endtime = System.currentTimeMillis();
        System.out.println("普通的反射: " + (endtime - starttime) + "ms");

    }

    public static void test03() throws NoSuchMethodException, InvocationTargetException, IllegalAccessException {
        User user = new User();
        Class c1 = user.getClass();
        Method getName = c1.getMethod("getName", null);
        long starttime = System.currentTimeMillis();

        getName.setAccessible(true);

        for (int i = 0; i < 1000000000; i++) {
            getName.invoke(user, null);
        }
        long endtime = System.currentTimeMillis();
        System.out.println("关闭检查的反射: " + (endtime - starttime) + "ms");

    }
}

在我的电脑上输出：

正常调用方法: 2ms
普通的反射: 1061ms
关闭检查的反射: 749ms

可以看到，虽然反射比直接调用慢了几百倍，但关闭权限检查可以提高一点点效率。

创建对象

Class c1 = Class.forName("User");

首先要用Class类创建一个Class对象，用newInstance()方法就能实例化，但是这种方法只能调用类的无参构造器，如果没写无参构造就会报错。

Object o = c1.newInstance();

由于知道是User类的实例，用的是User类的构造器，所以可以向下转型：

User user = (User)c1.newInstance();

要怎么用有参构造来实例化类呢？可以用Constructor：

  1. 通过getDeclaredConstructor()附加参数获取某个有参构造器
  2. 用构造器的newInstance()方法就可以创建实例了
Constructor constructor = c1.getDeclaredConstructor(int.class, int.class, String.class);
User yhz = (User)constructor.newInstance(001, 18, "yhz");

执行方法

基于接口定义了很多不同的类之后，由于反射是动态的，一般写的时候不知道这个实例化出来的对象是哪个类的，所以需要先获取Method对象用invoke()方法执行。

Object invoke (Object obj, Object… args)

invoke()方法第一个参数是用于哪个对象，后面的参数是这个方法需要传进去的参数。

Object对应原方法的返回值，若原方法无返回值则返回null。

若原方法为静态方法，此时Object obj可为null。

若原方法形参列表为空，此时Object[] args可为null。

若原方法声明为private，则需要在调用invoke前显式调用方法对象的setAccessible(true)方法。

Method setName = c1.getDeclaredMethod("setName", String.class);
Method getName = c1.getDeclaredMethod("getName",null);
setName.invoke(yhz,"YHZ");
System.out.println(getName.invoke(yhz,null));

会打印“YHZ”，打印的方法也可以直接通过重写toString来实现。

操作属性

获取到属性后，用get()方法和set()方法来操作属性。

在操作private属性时，需要用setAccessible(true)关闭权限检查。

Field name = c1.getDeclaredField("name");
name.setAccessible(true);
name.set(yhz,"Octopus");
System.out.println(name.get(yhz));

打印出了”Octopus”。

反射操作泛型

泛型类似于c++中的templatelist<int>，但它仅仅是给编译器javac使用的，确保数据的安全性和免去强制类型转化问题，但是，一旦编译完成，所有和泛型有关的类型全部擦除。

为了通过反射操作这些类型，Java新增了几种类型来代表不能被归一到Class类中的类型但是又和原始类型齐名的类型：

  • ParameterizedType：表示参数化类型，泛型的信息，比如Collection<String>
  • GenericArrayType：表示一种元素类型是参数化类型或者类型变量的数组类型
  • TypeVariable：是各种类型变量的公共父接口
  • WildcardType：代表一种通配符类型表达式

来试一试：

先定义两个函数

public void test01(Map<String, User> map, List<User> list) {
    System.out.println("test01");
}
public Map<String,User> test02(){
    System.out.println("test02");
    return null;
}

获得参数信息

public static void main(String[] args) throws NoSuchMethodException {
        Method method = Test04.class.getMethod("test01", Map.class, List.class);
		//上面两个方法和main方法都写在Test04.java文件里
        Type[] genericParameterTypes = method.getGenericParameterTypes();
        for (Type genericParameterType : genericParameterTypes) {
            System.out.println(genericParameterType);
        }
    }

用Method对象的getGenericParameterTypes()方法来打印参数类型。

输出：

java.util.Map<java.lang.String, User>
java.util.List<User>

但是这还不够，我们不想要Map和List，我们想要String和User！

所以改一改原来的代码，判断一下如果genericParameterType还是参数化类型，就得先把它转换成ParameterizedType再用getActualTypeArguments()获得里面的类型，再输出。虽然很麻烦，但是好在IDEA一下子就能帮忙写完：

public static void main(String[] args) throws NoSuchMethodException {
    Method method = Test04.class.getMethod("test01", Map.class, List.class);
    //上面两个方法和main方法都写在Test04.java文件里
    Type[] genericParameterTypes = method.getGenericParameterTypes();
    for (Type genericParameterType : genericParameterTypes) {
        System.out.println("#" + genericParameterType);
        if (genericParameterType instanceof ParameterizedType) {
            Type[] actualTypeArguments = ((ParameterizedType) genericParameterType).getActualTypeArguments();
            for (Type actualTypeArgument : actualTypeArguments)
                System.out.println(actualTypeArgument);
        }
    }
}

输出：

#java.util.Map<java.lang.String, User>
class java.lang.String
class User
#java.util.List<User>
class User

挺好的。

获得返回值

一样的套路，只不过是改一改方法，用getGenericReturnType()：

Method method1 = Test04.class.getMethod("test02");
Type genericReturnType = method1.getGenericReturnType();
if (genericReturnType instanceof ParameterizedType) {
	for (Type actualTypeArgument : ((ParameterizedType) genericReturnType).getActualTypeArguments()) {
		System.out.println(actualTypeArgument);
	}
}

输出：

class java.lang.String
class User

反射操作注解

  • getAnnotations
  • getAnnotation

先看个例子：

在写数据库时，需要建立对象关系映射（ORM，Object Relationship Mapping）。把对象变成表格

Class Student{
	int id;
	String name;
	int age;
}

会变成：

id name age
001 yhz 18
002 Octopus 18

要求：

  • 类和表对应
  • 属性和字段对应
  • 对象和记录对应

就可以利用注解和反射来做这个东西。先写几个注解吧：

//类名的注解
@Target(ElementType.TYPE)
@Retention(RetentionPolicy.RUNTIME)
public @interface Table {
    String value();
}
//属性的注解
@Target(ElementType.FIELD)
@Retention(RetentionPolicy.RUNTIME)
public @interface DBField {
    String columnName();
    String type();
    int length();
}

然后就可以把类加上注解了：

@Table("db_student")
public class Student {

    @DBField(columnName = "db_id",type = "int",length = 10)
    private int id;
    @DBField(columnName = "db_age",type = "int",length = 10)
    private int age;
    @DBField(columnName = "db_name",type = "varchar",length = 3)
    private String name;

    public Student() {
    }

    public Student(int id, int age, String name) {
        this.id = id;
        this.age = age;
        this.name = name;
    }

    public int getId() {
        return id;
    }

    public void setId(int id) {
        this.id = id;
    }

    public int getAge() {
        return age;
    }

    public void setAge(int age) {
        this.age = age;
    }

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    @Override
    public String toString() {
        return "Student{" +
                "id=" + id +
                ", age=" + age +
                ", name='" + name + '\'' +
                '}';
    }
}

public static void main(String[] args) throws ClassNotFoundException, NoSuchFieldException {
    Class student = Class.forName("Student");
    //getAnnotation
    for (Annotation annotation : student.getAnnotations()) {
        System.out.println(annotation);
    }
    //getAnnotationValue
    Table table = (Table)student.getAnnotation(Table.class);
    String value = table.value();
    System.out.println(value);
    
    //getFieldAnnotation
    Field name = student.getDeclaredField("name");
    DBField annotation = name.getAnnotation(DBField.class);
    System.out.println(annotation.columnName());
    System.out.println(annotation.type());
    System.out.println(annotation.length());

}

用getAnnotations()打印annotation对象，返回

@Table(value=db_student)

这个显然是不够的，我们要的是value里面的东西，所以应该获取具体的某个注解，要强制类型转换：

Table table = (Table)student.getAnnotation(Table.class);
String value = table.value();
System.out.println(value);

这样就能打印出value的值了

db_student

System.out.println(annotation.length());
