Kotlin速通日志

Android Kotlin
Programming
发布日期:   2022-02-09

Hello, World

fun main() {
    println("Hello Kotlin!")
}

Kotlin的语句不需要分号。

Kotlin基础

变量

Kotlin中定义变量的方式和Java区别很大，Kotlin有一套类型推导机制，所以定义变量只用两种关键字：

  • val（value）用来声明一个不可变的变量，对应Java中的final
  • var（variable）用来声明一个可变的变量，这种变量可以重新赋值

这里和python不一样的是，变量的类型在它第一次被赋值时决定，而且不能变更。如果第一次赋值是整数，那么这就是一个Int变量。

fun main() {
    val a = 10
    println(a)
}

也可以显式声明它的类型（最好这样用来避免错误）：

fun main() {
	var a: Int = 10
	a = a * 10
	println(a)
}

Kotlin中的类型全都是大写，可以看出它完全舍弃了Java的类型关键字，转而用对象数据类型。

Kotlin中的var和val的设计是用于解决Java中的final关键字不经常被使用的问题。现在应该先用val来定义变量，如果val不能满足需求，再改成var。

函数

fun methodName(param1: Int, param2: Int): Int {
    ...
    return 0
}

注意函数的返回值定义在参数列表后面。

简化

Kotlin有个语法糖，可以帮忙简化函数。

import kotlin.math.max

fun largerNumber(num1: Int, num2: Int): Int {
    return max(num1, num2)
}

当一个函数只有一行代码的时候，Kotlin允许不编写函数体，将唯一一行代码写在函数尾部，由于Kotlin的类型推导机制，这样子的包装另一个函数的函数返回值甚至不需要声明：

fun largerNumber(num1: Int, num2: Int) = max(num1, num2)

条件语句

if

Kotlin中的if和Java中的if没啥区别，特殊之处在于它可以有返回值。

fun largerNumber(num1: Int, num2: Int): Int {
    var value = 0
    if (num1 > num2) {
        value = num1
    } else {
        value = num2
    }
    return value
}

这段代码可以简化为

fun largerNumber(num1: Int, num2: Int) = if (num1 > num2) num1 else num2

if语句使用每个条件的最后一行来当作返回值。

when

when有点像switch，但是比switch好用。

Java中的switch只能传入字符串、短于整形或者整形变量作为条件，而且要加break，when就解决了这些问题。

fun getScore(name: String) = if (name == 'Tom'){
	86
} else if (name == 'Jim') {
	77
} else if (name == 'Jack') {
	95
} else if (name == 'Lily') {
	100
} else {
	0
}

有很多if...else的时候，就该使用when了：

fun getScore(name: String) = when (name) {
	"Tom" -> 86
    "Jim" -> 77
    "Jack" -> 95
    "Lily" -> 100
    else -> 0
}

when也是有返回值的。它允许传入一个任意类型的参数，然后可以用匹配值 -> { 执行逻辑 }来定义条件。

除了精确匹配之外，when还允许类型匹配：

fun checkNumber(num: Number) {
	when(num) {
		is Int -> println("Int")
		is Double -> println("Double")
		else -> println("else")
	}
}

is相当于Java中的instanceof关键字，由于函数接受了一个Number类型的参数，这是Kotlin内置的一个抽象类，与数字相关的类都是它的子类。

when还有一种不带参数的用法，可以自定义规则：

fun getScore(name: String) = when {
	name.startsWith("Tom") -> 86
    name == "Jim" -> 77
    name == "Jack" -> 95
    name == "Lily" -> 100
    else -> 0
}

所有名字以Tom开头的人的成绩都是86分。注意到Kotlin中判断对象相等可以用==，而不是Java中的equals方法。

循环

while循环和C一样用。

for

传统的for..i循环不能用，必须用for...in。

Kotlin可以这样生成一个闭区间：

val range = 0..10

有了区间之后，就可以用for来遍历

fun main() {
	for (i in 0..10) {
		println(i)
	}
}

用until来生成一个左闭右开区间，步长为2：

fun main() {
	for (i in 0 until 10 step 2) {
		println(i)
	}
}

用downTo来创建一个降序闭区间，也可以用step来确定步长：

fun main() {
	for (i in 10 downTo 1) {
		println(i)
	}
}

面向对象编程

类与对象

定义类与Java一致。但是实例化对象的时候不用new，只要调用构造函数即可：

val p = Person()

继承和构造函数

Kotlin里非抽象类默认是不能被继承的，这个设计的理由和val差不多，因为类和变量一样，最好都是不可变的，如果一个类允许被继承，那么它无法预知子类如何实现，可能会存在风险。

想让一个类可被继承，要加上open关键字：

open class Person {
    ...
}

要继承一个类，Java里面用extends，Kotlin里面用冒号：

class Student : Person() {
	...
}

为什么要调用Person的构造函数？

和Java不同的是，Kotlin里面把构造函数分为主构造函数和次构造函数。

主构造函数是最常用的构造函数，每个类默认有一个无参主构造函数，也可以指明参数，它的特点是没有函数体，直接定义在类名后面：

class Student (val sno: String, val grade: Int) : Person() {
    ...
}

这就表明在实例化的时候必须要传参，不然就会报错。

但是这样，主构造函数里面没有函数体，要是想在主构造函数里面加一些逻辑，就得用init代码块：

class Student (val sno: String, val grade: Int) : Person() {
    init {
    	println("Main Constructor")
    }
}

Java有规定：子类的构造函数一定要调用父类的构造函数，以前是类构造器帮忙做了这件事，现在要自己写出来。由于主构造函数没有函数体！虽然可以在init里面写，但是众所周知，程序员的懒狗特性使得Kotlin用继承类后面加括号来调用父类的构造函数。所以要调用Person的构造函数，就得在继承的时候写上括号，还可以指定调用某个有参构造函数：

open class Person(val name: String, val age: Int){
    ...
}
//另一个文件
class Student(val sno: String, val grade: Int, name: String, age: Int) : Person(age, name) {
    ...
}

由于要给Person类的构造函数传入name和age两个字段，但是没有接受参数，所以在Student类的构造函数里面必须写上这两个参数，而且不能把它们声明成val或者var，因为在主构造函数里面声明成val和var的变量会自动成为这个类的字段，导致和父类重名的age和name字段冲突，如果不加val和var，它们就是两个局部变量，作用域仅在主构造函数中，这样就达到了不新增字段，还能给构造函数传参的目的。

现在可以这样来创造实例了：

val student = Student("a123", 5, "Jack", 19)

但是括号不止于此！因为还有一种次构造函数

任何类都只能有一个主构造函数，在类名处定义。但是一个类可以有多个次构造函数，次构造函数也可以拿来实例化一个类，和主构造函数没啥不一样，只是它有函数体。

Kotlin规定，当一个类既有主构造函数又有次构造函数的时候，所有的次构造函数都必须调用主构造函数（包括间接调用）：

class Student(val sno: String, val grade: Int, name: String, age: Int) : Person(name, age) {
    constructor(name: String, age: Int) : this("", 0, name, age) {
    }
    constructor() : this("", 0){
    }
}

现在有三个构造函数：传入四个参数的主构造函数，传入两个参数的次构造函数（调用了主构造函数），无参次构造函数（调用了传入两个参数的次构造函数）。

次构造函数用constructor关键字定义，第一个次构造函数接受name和age参数，然后又通过this关键字调用了主构造函数，并将sno和grade这两个参数赋初值。第二个次构造函数不接受任何参数，它通过this调用了第一个次构造函数，把所有参数都赋值成初始值，由于第二个次构造函数间接调用了主构造函数，所以也是允许的。

Kotlin也允许类里面没有主构造函数，只有次构造函数：

class Student : Person {
	constructor(name: String, age: Int) : super(name, age) {
	}
}

由于没有主构造函数，也不用调用父类的构造函数了，所以Person后面就不用加括号了。而且次构造函数只能调用父类的构造函数了。

接口

Kotlin的接口和Java的基本上相同。

interface Study {
    fun readbooks()
    fun doHomework()
}

然后就可以让Student类去实现接口：

class Student(name: String, age: Int) : Person(name, age), Study{
    override fun readbooks() {
        println(name + "is reading")
    }
    override fun doHomework() {
        println(name + "is doing homework")
    }
}

Java中继承的关键字是extends，实现接口使用的是implements，但是Kotlin全都改成了冒号，中间用括号分割。接口后面不用加括号，因为接口没有构造函数。

Kotlin现在可以对接口内函数写默认功能了！只要在接口里面写就好。这样就不会强制要求实现写好的函数了。

修饰符

private和public的作用和在Java里一样，但是现在默认值变成public了。

protected

Java中表示对当前类、子类和同一包路径下的类可见

Kotlin中表示只对当前类和子类可见

internal

Kotlin中没有Java的default可见性（同一包路径下的类可见），转而使用internal，只对同一模块下的类可见，用来打包模块应该会很好用。

数据类

数据类用于将服务器端或者数据库中的数据映射到内存中，提供数据模型支持。MVC “Model-View-Controller” 里的M就是数据类（Model）。

数据类需要重写几个方法：equals()、hashCode()、toString()，以对统一使用提供更好的支持。

Kotlin的数据类比起Java可谓是大幅度删减，Java中要这样写：

public class Cellphone {
	String brand;
    double price;
    
    public Cellphone(String brand, double price) {
        this.brand = brand;
        this.price = price;
    }
    
    @override
    public boolean equals(Object obj) {
        if(obj instanceof Cellphone) {
            Cellphone other = (Cellphone)obj;
            return other.brand.equals(brand) && other.price == price;
        }
        return false;
    }
    
    @override
    public int hashCode() {
        return brand.hashCode() + (int)price;
    }
    
    @override
    public String toString() {
        return "Cellphone(brand=" + brand + " ,price=" + price + ")";
    }
}

其实就是定义了两个字段 ，写了一个构造函数，然后重写了上述的方法而已。

所以在Kotlin中可以这样写（一个类里面没有代码的时候就不用写大括号）：

data class Cellphone(val brand: String, val price: Double)

好！data关键字一写上去，编译器就知道你希望这个类是数据类，于是就会帮你根据构造函数里面写的参数自动生成方法，省下按alt+insert的功夫。

单例类

单例模式（Singleton Pattern）是最基础的设计模式之一，用于避免创建重复的对象，也就是某个类在全局最多有一个实例。最常见的写法是这样的：

public class Singleton {
    private static Singleton instance;
    
    private Singleton() {}
    
    public Synchronized static Singleton getInstance() {
        if(instance == null) {
            instance = new Singleton();
        }
        return instance
    }
    
    public void test() {
        System.out.println("test is called");
    }
}

解释一下Synchronized，一个线程访问一个对象中的synchronized(this)同步代码块时，其他试图访问该对象的线程将被阻塞。

那么这段代码就很好理解了：为了防止外部创建Singleton的实例，要写一个private构造函数，然后给外部提供一个getInstance()的静态方法来获取一个实例。要么创建一个新的实例并返回，要么就直接返回已有实例。

如果想调用单例类里面的方法：

Singleton singleton = Singleton.getInstance();
singleton.test();

如果想打破单例模式，用反射获取私有构造器。

Kotlin提供的单例类做了进一步的解耦和封装，把实现逻辑隐藏起来，只要拿去用就可以了。想创建一个单例类，只要把class改成object：

object Singleton() {
    fun test() {
        println("test is called")
    }
}

把class改成object，写一些函数，轻松又愉快。

调用方式类似于静态方法：

Singleton.test()

调用这个方法，会自动生成一个实例，并且保证全局只存在一个实例。

函数式编程

集合的创建和遍历

用的比较多的集合大概是List、Set、Map这三个接口，List用ArrarList和LinkedList实现，Set用HashSet实现，Map用HashMap来实现。

想创建一个列表，比起Java要先创建一个ArrayList的实例，再一个个添加元素，Kotlin提供内置的listOf()函数，用来初始化集合（当然老方法也能用）：

val list = listOf("Apple", "Banana", "Orange")

用for循环遍历集合：

for (fruit in list) {
	println(fruit)
}

但是listOf创建的是元组，没法改，这点的设计思路大概和可继承类、常量和变量的思路是一样的。

要创建一个可变集合，用mutableListOf()：

val list = mutableListOf("Apple", "Banana", "Orange")
list.add("Grape")
for (fruit in list) {
    println(fruit)
}

set集合的用法差不多，把名字换成setOf()和mutableSetOf()就行了。

集合和列表的区别就是集合不能存放重复元素。

最后是Map，它的用法不一样，Java式的写法是这样的：

val map = HashMap<String, Int>()
map.put("Apple", 1)
map.put("Banana", 2)
map.put("Orange", 4)

但是Kotlin不太建议用put/get，它重载了[]：

map["Apple"] = 1
val num = map["Apple"]

它也提供了mapOf()和mutableMapOf()，这里的to是infix函数，不是关键字：

val map = mapOf("Apple" to 1, "Banana" to 2, "Orange" to 4)
for ((fruit, number) in map) {
    println("fruit: " + fruit + ", number: " + number)
}

集合之函数式API

这里用到很多Lambda表达式。Lambda是一段可以作为参数传递的代码，也就是匿名函数，语法结构是这样的：

{参数列表 -> 函数体}

参数列表大概是“参数名1: 参数类型, 参数名2: 参数类型”这样声明的。函数体中可以编写任意长度的代码，最后一行会作为返回值（一般只写一行）。

举个例子：想找出列表里面最长的单词再也不用写好几行了，只要用list类的maxBy()方法，这个方法接收一个Lambda参数，遍历集合并把每次遍历的值作为参数传给Lambda表达式，找到返回值的最大值对应的元素。有点像map()。

val list = listOf("Apple", "Banana", "Orange")
val lambda = { str: String -> str.length }
val maxLengthFruit = list.maxBy(lambda)
// maxLengthFruit存的是水果，不是长度

lambda接受一个字符串参数，返回字符串的长度。

有四个特性可以用来简化：

  1. 当lambda参数是函数的最后一个参数时，可以把表达式移到括号外面() {}
  2. 如果lambda参数是唯一参数，可以省略括号{}
  3. 由于Kotlin的类型推导机制，参数列表大多数情况下不用写类型
  4. 当lambda函数的参数列表只有一个参数的时候，也不用写参数列表了，直接用it关键字就好
val list = listOf("Apple", "Banana", "Orange")
val maxLengthFruit = list.maxBy{ it.length }

虽然简化的很彻底，但是代码的意思还是清晰易懂，见名知意。

集合的函数式API里面也有下面的两个函数，之前在Python里面写过：

map()函数，它接收一个lambda参数，将集合中的每个元素映射成另一个集合。

val list = listOf("Apple", "Banana", "Orange")
val newlist = list.map{ it.toUpperCase() }

这个操作把list里面的水果变成大写。

filter()函数用来过滤集合中的数据，可以单独用，也可以和map()一起用：

val list = listOf("Apple", "Banana", "Orange")
val newList = list.filter { it.length < 5 }.map { it.toUpperCase() }

如果先用map再用filter，效率会变低，因为做了无用功。

还有两个API：any和all函数

any用于判断集合中是否至少有一个元素符合条件，all用于判断集合中是否所有元素都符合条件：

val list = listOf("Apple", "Banana", "Orange", "Watermelon")
val anyResult = list.any { it.length <= 5 }
val allResult = list.all { it.length <= 5 }

显然any的结果是True，all的结果是False。

Java函数式API

在Kotlin调用接收单抽象接口参数的Java方法时也可以用函数式API。单抽象接口参数指的是接口里只有一个待实现方法。

如Runnable接口：

public interface Runnable {
	void run();
}

任何一个Java方法，只要它接收Runnable参数，就可以用函数式API。

Thread类的构造方法里接收了一个Runnable参数：

//java.lang.Thread
public Thread(Runnable target) {
        init(null, target, "Thread-" + nextThreadNum(), 0);
    }

于是可以这样用：

new Thread(
    new Runnable() {
		@override
		public void run() {
			System.out.println("Thread is running");
		}
	}
).start()

这是个匿名类的写法，创建一个Runnable接口的匿名类实例，再把它传给Thread类的构造方法，最后调用Thread类的start方法。

这是Kotlin版本：

Thread(Object : Runnable {
	override fun run(){
		println("Thread is running")		
	}
}).start()

Kotlin中由于没有new关键字，所以创建匿名类实例的时候用object关键字。

用Lambda参数来简化一下：

Thread(Runnable { println("Thread is running") }).start()

因为Runnable类里面只有一个待实现方法，所以编译器会很聪明的认为Runnable后面的Lambda表达式重写了run方法。

如果一个Java方法的参数列表里面有且仅有一个单抽象方法接口参数，还能省略接口名字，因为编译器知道你传的是这个类型的参数了：

Thread { println("Thread is running") }.start()

简直就是神奇的西方魔法！

这里的Java函数式API限定于从Kotlin调用Java方法，因为Kotlin里面有更好用的自定义函数式API，不需要Java这样借助单抽象方法接口来实现捏。

空指针检查

NullPointError是出现次数最多的错误！编译器是检查不出来这种错误的，只能通过逻辑判断来避免。

如果给某个函数传null，就可能出现空指针错误。所以应该加个检查：

public void doStudy(Study study) {
    if(study != numm)
		study.readbook();
}

可空类型系统

Kotlin编译时有做检查，让空指针错误比较难发生

fun doStudy(study: Study) {
	study.readbook()
}

因为Kotlin默认所有的参数和变量都不为空，所以这里传入的参数也不能为空（如果为空，会报错）
但是这样又出现了新的问题，万一某些应用需要某个参数为空怎么办？

Kotlin有一套可空类型系统，在用可空类型系统时，就要人工处理空指针了。

可空类型系统只要在类名后面加一个问号

fun doStudy(study: Study?) {
    if (study != null)
		study.doHomework()
}

如果不判断null，编译器会报错。
